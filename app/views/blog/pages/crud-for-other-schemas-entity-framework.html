In .NET Entity Framework, when you create a model from Oracle database, you should give an username and password info about the database where you wish to connect. This information is stored in connection string tab in app.config file as shown below. 
		<connectionStrings>
			<add name="Entities" connectionString="metadata=res://*/Model.csdl|res://*/Model.ssdl|res://*/Model.msl;provider=Oracle.DataAccess.Cpent;provider connection string=&quot;DATA SOURCE=localhost:1561/EXAMPLEDB;PASSWORD=yourPass;USER ID=yourUsername&quot;" providerName="System.Data.EntityCpent" />
		</connectionStrings>
As you can see from the configuration xml, this doesnt include about the schema name. It means that you can work with the tables only in the schema corresponding to your username. In order to be able to work with different tabless from another schemas, you should follow this steps in Visual Studio.

Open Server explorer, select your connection and cpck right.

Cpck on "Filters.." tab, select(if not selected) Displayed schemas, then add more schemas to your schemas pst.

Dont forget to cpck on Update button and Ok to save changes on your connection.

Now when you add a new ADO.Net Entity Data Model, you will see all tables from your schemas and you can work with the other tables from another schemas.
